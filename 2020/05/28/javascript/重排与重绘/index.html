浏览器渲染原理

发表于 2020-05-28 | 分类于 JavaScript

从用户在浏览器地址栏输入网址，到看到整个页面，中间发生了什么？中间发生的大体分为三个阶段：

  • HTTP请求阶段

  • HTTP响应阶段

  • 浏览器渲染阶段

涉及到的理论：

  • 一个进程（process）里可以有多个线程（thread），线程是重要干活的

DOM的重绘（Repaint）和回流（Reflow）

重绘： 元素样式的改变（但宽高、大小、位置等不变）

如outline、visibility、color、background-color等

回流：元素的大小或者位置发生了变化（当页面布局和几何信息发生变化的时候），触发了重新布局，导致渲染树重新计算布局和渲染。

如：

  • 添加或者删除可见的DOM元素；
  • 元素的位置发生变化；
  • 元素的尺寸发生变化；
  • 内容发生变化（比如文本变化或者图片被另一个不同尺寸的图片所替代）；
  • 页面一开始渲染的时候（这个无法避免）；
  • 因为回流是根据视口的大小来计算元素的位置和大小的，所以浏览器的窗口尺寸变化也会引发回流。

回流一定会触发重绘，而重绘不一定会回流。

性能优化：避免DOM的回流

  • 分离读写操作（现代的浏览器都有渲染队列的机制）

js中操作盒模型的属性：offsetTop、offsetLeft、offsetWidth、offsetHeight、clientTop、clientLeft、clientWidth、clientHeight、scrollTop、scrollLeft、scrollWidth、scrollHeight、getComputedStyle、currentStyle
